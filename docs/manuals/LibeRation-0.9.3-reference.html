Configure a continuous-time hidden Markov model

Description

'nm_cthmm_config()' is the continuous-time counterpart of [nm_hmm_config()]. Off-diagonal entries of 'generator' name transition-rate assignments in '$ERROR'; diagonal entries are generated as minus the sum of the other entries in their row. At every observation the compiled engine calculates 'exp(Q * DT)' with a CppAD-compatible matrix exponential. This supports irregular observation times, time-varying rates, exact gradients, retrospective smoothing, and Viterbi decoding.

Usage

nm_cthmm_config(
  states,
  initial,
  generator,
  emission,
  initial_scale = c("probability", "log"),
  rate_scale = c("rate", "log"),
  emission_scale = c("likelihood", "log"),
  by_dvid = TRUE
)

Arguments

states

Unique labels for the hidden states.

initial

Character vector naming one '$ERROR' initial-state assignment per state.

generator

Square character matrix. Every off-diagonal element names the '$ERROR' assignment for the corresponding from-state to to-state rate. Diagonal elements are ignored and may be empty or 'NA'.

emission

Character vector naming one state-conditional '$ERROR' emission assignment per state.

initial_scale

Whether 'initial' contains probability weights or log weights.

rate_scale

Whether off-diagonal generator assignments contain non-negative rates or unconstrained log rates.

emission_scale

Whether 'emission' contains likelihoods/densities or log likelihoods.

by_dvid

Maintain independent state sequences for each 'DVID' within subject.

Value

A validated, serializable continuous-time hidden Markov configuration.

Estimate a population pharmacometric model

Description

Deterministic conditional methods use exact CppAD gradients for ETA modes. LAPLACE uses the exact conditional Hessian; FOCE/FOCEI use the interaction-aware Gauss-Newton curvature; FO integrates the first-order Gaussian linearization analytically. GQ integrates the exact joint objective over ETAs with adaptive tensor or Smolyak sparse Gauss–Hermite quadrature. Stochastic methods use the same C++ joint objective and are implemented in the stochastic estimation module. HMC and NUTS use exact joint CppAD gradients with unconstrained parameter transforms, dual-averaged step-size adaptation, and a diagonal mass matrix. NPML and NPAG replace the Gaussian random-effect distribution with a discrete ETA support distribution; NPAG adapts, expands, and prunes that support. Serial FO, FOCE, FOCEI, Laplace, and ITS fits expose a persistent C++ population objective through thin callbacks to R's L-BFGS-B/BFGS optimizer; PSOCK fits retain R coordination across their persistent C++ workers.

Usage

nm_est(
  model,
  data,
  method = c("FOCEI", "FOCE", "FO", "LAPLACE", "ITS", "GQ", "IMP", "SAEM", "BAYES",
    "HMC", "NUTS", "NPML", "NPAG"),
  maxit = 200L,
  eta_maxit = 100L,
  tolerance = 1e-06,
  trace = 0L,
  print_every = 0L,
  n_cores = 1L,
  optimizer_backend = c("auto", "native", "r"),
  covariance = FALSE,
  covariance_type = c("auto", "hessian", "opg", "sandwich", "r", "s"),
  covariance_tolerance = 1e-08,
  covariance_samples = NULL,
  covariance_seed = NULL,
  initial_eta = NULL,
  collect_output = TRUE,
  ...
)

Arguments

model

An [nm_model()] or compiled [NMEngine].

data

A NONMEM-style event dataset.

method

Estimation method.

maxit

Maximum outer evaluations.

eta_maxit

Maximum conditional ETA iterations.

tolerance

Relative optimization tolerance.

trace

Optimizer tracing level.

print_every

Write the objective and scaled population gradient to stdout every N outer evaluations. Zero disables iteration logging.

n_cores

Number of parallel subject workers. Parallel workers persist compiled C++ engines for the duration of the estimation.

optimizer_backend

Use the experimental compiled scaled, bounded BFGS optimizer (‘"native"'), R’s mature L-BFGS-B/BFGS driver ('"r"'), or the calibrated production policy (‘"auto"'). Auto selects R’s driver while its objective, gradient, conditional-mode state, parameter transforms, and covariance derivatives remain in the persistent C++ population evaluator.

covariance

Run a covariance step after estimation.

covariance_type

Covariance estimator: automatic R/S fallback, objective Hessian ('"hessian"'), subject-score OPG ('"opg"'), or robust sandwich ('"sandwich"'). '"r"' and '"s"' are accepted aliases.

covariance_tolerance

Positive-definite regularization tolerance for the covariance step.

covariance_samples

Target integration budget for an IMP or SAEM covariance step. Low-dimensional ETA integrals use tensor Gauss–Hermite quadrature near this budget; higher-dimensional integrals use random-normal importance samples. The default reuses the IMP count or uses 200 for SAEM.

covariance_seed

Common-random-number seed for the random-normal covariance fallback. The default reuses the estimation seed.

initial_eta

Optional finite subject-by-ETA matrix used to warm-start compatible conditional or SAEM estimation steps.

collect_output

Whether selected generated OUTPUT columns should be evaluated and retained with the completed fit.

...

Method-specific controls. For 'method = "GQ"', use 'gq_grid' ('"auto"', '"tensor"', or '"smolyak"'), 'gq_order' (tensor nodes per ETA dimension, default 5), 'gq_level' (Smolyak level, default 3), 'gq_adaptive' (default 'TRUE'), 'gq_max_points' (retained-grid allocation guard, default 100000), and 'gq_gradient' ('"score"' or the slower '"finite_grid"'). Automatic selection uses tensor quadrature through three ETAs and Smolyak quadrature for higher-dimensional models. For 'method = "HMC"' or '"NUTS"', controls include 'n_warmup' (500), 'n_sample' (1000 per chain), 'n_thin' (1), 'n_chains' (4), 'seed', optional 'step_size', 'target_acceptance' (0.8), 'adapt_mass' ('TRUE'), 'n_leapfrog' (10; HMC), 'max_depth' (10; NUTS), and 'divergence_threshold' (1000). 'sampler_backend = "native"' keeps complete trajectories in C++; '"r"' retains the slower reference implementation for numerical comparison. For 'method = "NPML"' or '"NPAG"', use 'np_supports' for an optional fixed starting matrix, 'np_points' (25), 'np_max_support' (100), 'np_min_weight' (1e-5), 'np_weight_maxit' (1000), 'np_cycles' (3), and, for NPAG, 'np_grid_step' (1), 'np_grid_decay' (0.5), and 'np_max_candidates' (500). 'np_estimate_population' controls alternating THETA/SIGMA updates. Ordinary covariance is not regular for a discrete support distribution; use bootstrap uncertainty for NPML/NPAG.
