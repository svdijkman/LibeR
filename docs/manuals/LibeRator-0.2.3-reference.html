Longitudinal Bayesian patient assessment

Description

Each call creates an immutable assessment tied to hashes of its patient evidence, model, endpoint, covariate policy, and cutoff. 'dynamic' mode fits occasion-specific ETA states under a Gaussian random-walk prior while the compartment state remains continuous across boundaries.

Usage

lator_assess(
  patient,
  model,
  endpoint,
  analyte = endpoint$drug,
  cutoff = Inf,
  mode = c("static", "dynamic"),
  state_times = NULL,
  covariate_policies = list(),
  process_scale = 0.1,
  process_covariance = NULL,
  workspace = NULL,
  actor = "local-session",
  ...
)

Arguments

patient

Longitudinal patient record.

model

LibeRation population model.

endpoint

Versioned therapeutic endpoint.

analyte

Event drug/analyte name; defaults to 'endpoint$drug'.

cutoff

Patient-timeline hour through which evidence is visible.

mode

'"static"' or '"dynamic"' latent patient parameters.

state_times

Optional dynamic-state boundaries; otherwise explicit 'state_boundary' events are used.

covariate_policies

Named per-covariate policy lists passed to [lator_covariate_at()].

process_scale

Random-walk innovation covariance as a multiple of OMEGA.

process_covariance

Optional explicit innovation covariance.

workspace

Optional workspace. When supplied, the immutable assessment is appended to and saves the patient record.

actor

Audit actor used when saving.

...

Additional arguments to [LibeRation::nm_individual_fit()].

Resolve a longitudinal covariate at specified times

Description

Missing observations are never silently converted into population values. Carry-forward, interpolation, nearest-neighbour, and fallback behaviour are explicit policies and every returned value records its provenance and age.

Usage

lator_covariate_at(
  patient,
  name,
  times,
  method = c("locf", "linear", "nearest", "none"),
  max_age = Inf,
  max_gap = Inf,
  fallback = NULL,
  fallback_unit = "",
  uncertainty_sd = NA_real_,
  cutoff = Inf
)

Arguments

patient

A 'lator_patient'.

name

Covariate event name.

times

Numeric timeline hours.

method

One of '"locf"', '"linear"', '"nearest"', or '"none"'.

max_age

Maximum permitted age in hours for carried or nearest values.

max_gap

Maximum interval in hours spanning a linear interpolation.

fallback

Optional explicit fallback value. It is labelled 'fallback', never as an observation.

fallback_unit

Unit for 'fallback' if no observed unit is available.

uncertainty_sd

Optional standard deviation attached to fallback values. Event-specific 'metadata$uncertainty_sd' takes precedence for observed values.

cutoff

Ignore evidence recorded after this patient-timeline time.

Value

A provenance-rich data frame with one row per requested time.

Description

It is meant for software teaching and tests; its target range is not a real dosing rule.

Launch the LibeRator longitudinal dosing workbench

Description

By default the server binds only to loopback. Exposing it to another host is deliberately blocked unless 'allow_remote = TRUE'; production deployment additionally requires institutional authentication, TLS, authorization, backup, monitoring, validation, and clinical governance outside this research package.

Usage

lator_gui(
  workspace = NULL,
  path = NULL,
  passphrase = NULL,
  key = NULL,
  models = NULL,
  endpoints = NULL,
  host = "127.0.0.1",
  port = NULL,
  launch.browser = TRUE,
  allow_remote = FALSE,
  session_workspace = FALSE,
  teaching_example = FALSE
)

Arguments

workspace

Optional unlocked workspace.

path

Workspace path when 'workspace' is not supplied.

passphrase

Optional passphrase. If omitted, an unlock screen is shown.

key

Optional managed 32-byte key.

models

Session models in addition to encrypted registrations.

endpoints

Session endpoints in addition to encrypted registrations.

host, port, launch.browser

Passed to [shiny::runApp()].

allow_remote

Explicitly permit a non-loopback bind for governed test deployments.

session_workspace

Create a separate ephemeral encrypted workspace for every browser session. This is intended for hosted research demonstrations and prevents application users from sharing a workspace directory.

teaching_example

Seed an otherwise empty workspace with the synthetic AED teaching patient, model, and endpoint. Intended for demonstrations; the example is explicitly non-clinical.

Value

Invisibly, the Shiny app.

Compare candidate regimens under posterior uncertainty

Description

The complete dosing history is replayed for every posterior draw so that accumulated compartment amounts are retained. Parameter uncertainty is evaluated in one batched C++ simulation call per candidate.

Usage

lator_regimen_optimise(
  assessment,
  patient,
  candidates,
  endpoint = assessment$endpoint,
  nsim = 200L,
  grid_step = 0.25,
  start_time = NULL,
  residual = FALSE,
  max_daily_dose = Inf,
  max_single_dose = Inf,
  dose_burden_weight = 0,
  seed = NULL,
  n_cores = 1L
)

Arguments

assessment

A completed 'lator_assessment'.

patient

The corresponding patient timeline.

candidates

Candidate data frame from [lator_regimen_candidates()].

endpoint

Endpoint to optimise; defaults to the assessment endpoint.

nsim

Number of posterior draws.

grid_step

Prediction-grid spacing in hours.

start_time

First candidate dose time; defaults just after the latest historical event visible to the assessment.

residual

Include residual observation variability in attainment probabilities.

max_daily_dose

Optional hard feasibility constraint.

max_single_dose

Optional hard feasibility constraint.

dose_burden_weight

Small optional tie-breaker applied to daily dose.

seed

Reproducible seed.

n_cores

Simulation cores used within LibeRation.

Value

A ranked result with summaries and per-candidate trajectories.

Create an explicit future prediction for a selected regimen

Description

Regimen comparison already simulates every feasible candidate under posterior uncertainty. This function promotes one candidate trajectory into a separate, auditable forecast artifact without repeating the expensive simulation or treating the highest-ranked row as an automatic dosing decision.

Usage

lator_regimen_predict(
  comparison,
  candidate_id,
  probs = c(0.05, 0.5, 0.95)
)

Arguments

comparison

Result from lator_regimen_optimise().

candidate_id

Candidate deliberately selected by the user.

probs

Three ordered probabilities used for the lower interval, median, and upper interval.

Value

A lator_future_prediction containing pointwise uncertainty bands, the selected regimen, linked assessment and endpoint evaluation.

Open an encrypted LibeRator workspace

Description

Patient records, the patient catalogue, endpoints, model registrations and the audit chain are encrypted and authenticated with libsodium. The key is derived with Argon2 and is never written to the workspace. Direct patient identifiers are deliberately outside the schema; use a pseudonym generated by the controlling study or institution.

Usage

lator_workspace(path = NULL, passphrase = NULL, key = NULL, create = TRUE)

Arguments

path

Workspace directory.

passphrase

Session passphrase of at least 12 characters.

key

Optional 32-byte raw key for managed deployments.

create

Create a missing workspace.

Value

A key-bearing 'lator_workspace'. Do not serialize or transmit it.
