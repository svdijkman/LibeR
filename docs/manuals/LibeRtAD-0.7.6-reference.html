Benchmark the persistent C++ automatic-differentiation engine

Description

Separates recording from repeated tape evaluation and compares exact CppAD derivatives with central finite differences evaluated in R. The results are intended for local regression and hardware comparison rather than as a universal performance claim.

Usage

ad_benchmark(
  case = c("rosenbrock", "pk", "jacobian"),
  iterations = 1000L,
  warmups = 50L,
  optimize = TRUE,
  finite_difference = TRUE
)

Arguments

case

Built-in workload returned by ad_benchmark_cases().

iterations

Number of requested measured value and derivative calls. The timer may increase this count to obtain stable resolution.

warmups

Number of calls before timing.

optimize

Whether CppAD tape optimisation is enabled.

finite_difference

Include R central finite-difference comparators.

Value

An ad_benchmark_result containing timings, accuracy checks, engine metadata, and settings.

Examples

ad_benchmark_cases()
ad_benchmark("pk", iterations = 100, warmups = 5)

Benchmark nested-AD-safe CppAD checkpoint prototypes

Description

Compares repeated direct recordings with 'chkpoint_two' prototypes for an ADVAN1 interval and a 2-by-2 matrix state update. The checkpoint objects are explicitly configured for the nested AD route used by LibeRation curvature calculations. Results are diagnostic; production selection remains guarded by measured performance and tape size.

Usage

ad_checkpoint_probe(repetitions = 64L, evaluations = 1000L)

Arguments

repetitions

Number of repeated kernel uses on each outer tape.

evaluations

Number of zero-order evaluations used for timing.

Value

A named list of operation counts, accuracy checks, and timings.

Compile an R-like mathematical model

Description

Compile an R-like mathematical model

Usage

ad_compile(
  code,
  inputs = character(),
  outputs = NULL,
  at = NULL,
  wrt = names(at),
  optimize = TRUE
)

Arguments

code

Character model code. The established LibeRation assignment, 'THETA(i)', 'ETA(i)', 'SIGMA(i)', and 'ERR(i)' syntax is supported.

inputs

Optional external input names. Undeclared symbols are appended as inputs in first-use order.

outputs

Optional assignment names to expose. The default exposes all assignments, with the most recent value for reassigned names.

at

Optional named recording point. When supplied, a persistent tape is recorded immediately.

wrt

Inputs with respect to which derivatives are required.

optimize

Whether CppAD should optimize the recorded tape.

Value

An 'ADModel' R6 object.

Examples

model <- ad_compile(
  "CL = THETA(1) * exp(ETA(1))\nPENALTY = log(CL)^2",
  at = c(THETA_1 = 2, ETA_1 = 0),
  wrt = c("THETA_1", "ETA_1"),
  outputs = "PENALTY"
)
model$value_gradient(c(THETA_1 = 2, ETA_1 = 0))
model$hessian(c(THETA_1 = 2, ETA_1 = 0))

Standard-normal tensor Gauss–Hermite quadrature grid

Description

Constructs a tensor-product Gaussian quadrature rule for expectations under a multivariate standard-normal distribution. The one-dimensional rule uses the Golub–Welsch eigensystem in C++; tensor nodes and log weights are also assembled in C++ to avoid repeated R-side grid construction.

Usage

ad_gauss_hermite(order = 5L, dimension = 1L, max_points = 100000L)

Arguments

order

Number of nodes per dimension.

dimension

Dimension of the normal integral. Zero returns the single empty node with unit mass.

max_points

Maximum permitted tensor-grid size. This guard prevents an accidental exponential allocation as the dimension grows.

Value

A list containing 'nodes', normalized 'weights', 'log_weights', and grid metadata.

Standard-normal Smolyak Gauss–Hermite quadrature grid

Description

Constructs an isotropic Smolyak sparse grid for multivariate standard-normal expectations. Level 'i' uses the odd-order one-dimensional Gauss–Hermite rule with '2 * i - 1' nodes. Duplicate nodes are consolidated in C++ and negligible cancellation residues are removed.

Usage

ad_smolyak_gauss_hermite(level = 3L, dimension = 4L, max_points = 100000L)

Arguments

level

Sparse-grid accuracy level. Level one is the single origin node; increasing the level adds higher-order interactions.

dimension

Dimension of the normal integral. Zero returns the single empty node with unit mass.

max_points

Maximum permitted number of retained sparse-grid nodes.

Details

Smolyak combination weights are signed. 'weights' therefore contains the signed masses, while 'signs' and 'log_abs_weights' provide a numerically stable representation for likelihood integration.

Value

A list containing 'nodes', signed 'weights', 'signs', 'log_abs_weights', and sparse-grid metadata.

Pointer-backed compiled automatic-differentiation model

Description

'ADModel' is deliberately light: the serializable IR is retained for worker reconstruction, while C++ owns the executable program and persistent tape.

Public fields

ir

Serializable intermediate representation created by [ad_ir()].

program_ptr

External pointer to the compiled C++ expression program.

tape_ptr

External pointer to the persistent CppAD tape, or 'NULL' until [ADModel]$record() is called.

wrt

Character vector naming the active differentiation inputs.

outputs

Character vector naming the active model outputs.

dynamic

Character vector naming non-differentiated inputs retained as CppAD dynamic parameters.

dynamic_values

Current named values of the tape's dynamic parameters.

Launch the LibeRtAD benchmark laboratory

Description

Runs native tape microbenchmarks and, when the LibeR repository harness is available, launches the paired LibeRation/NONMEM workflow benchmark in a cancellable background process.

Usage

libertad_gui(
  benchmark_root = NULL,
  host = "127.0.0.1",
  port = NULL,
  launch.browser = TRUE,
  allow_remote = FALSE
)

Arguments

benchmark_root

Optional LibeR repository root or benchmark script path.

host, port, launch.browser

Passed to shiny::runApp().

allow_remote

Explicitly permit a non-loopback bind.

Value

Invisibly, the Shiny app.
