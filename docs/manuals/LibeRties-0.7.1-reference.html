Persistent cross-platform LibeR job queue

Description

Each user receives a separate filesystem namespace. Live jobs run in fresh 'callr' R processes and reconstruct C++ pointers from serialized models.

Public fields

root

Normalized persistent queue storage directory.

user

Safe tenant namespace used by default for queue operations.

max_workers

Maximum simultaneous background workers.

limits

Effective resource and storage limits.

isolation

Description of the worker isolation strategy.

processes

Internal environment of live 'callr' worker handles.

Authenticated multi-tenant LibeRties server core

Description

This object is the transport-independent server boundary used by a local test host and, later, the HTTP adapter. Every job operation derives the user namespace from the bearer token; callers cannot nominate another tenant.

Create the LibeRties server-administration application

Description

The browser session is protected by an administrator token whose digest is held only in the app process. It is deliberately separate from tenant API tokens. Use a long random value supplied through 'LIBERTIES_ADMIN_TOKEN'.

Usage

ls_admin_gui(
  root = .ls_default_root(),
  admin_token = Sys.getenv("LIBERTIES_ADMIN_TOKEN")
)

Arguments

root

LibeRties server storage root.

admin_token

Administrator login token. Defaults to the 'LIBERTIES_ADMIN_TOKEN' environment variable.

Value

A Shiny application object.

  root = .ls_default_root(),
  host = "127.0.0.1",
  port = 8000L,
  max_workers_per_user = 2L,
  quiet = FALSE,
  production = !.ls_loopback(host),

  isolation_probe = NULL
)

Arguments

root

Persistent server root.

host

Listen address.

port

TCP port.

max_workers_per_user

Host-level per-user worker ceiling.

quiet

Suppress Plumber startup messages.

production

Enforce production preflight. Defaults to 'TRUE' for a non-loopback host.

behind_tls_proxy

Confirm that HTTPS is terminated by a maintained reverse proxy.

policy

Optional security policy.

isolation_probe

Deployment-integrated isolation verifier passed to [ls_server_preflight()].

Define a LibeRties deployment-security policy

Description

Define a LibeRties deployment-security policy

Usage

ls_security_policy(
  production = FALSE,
  requests_per_minute = if (production) 120L else 10000L,
  max_rate_limit_keys = 10000L,
  trusted_proxies = character(),
  max_log_lines = 5000L,
  max_log_bytes = 1024^2,
  require_storage_encryption = production,
  require_os_isolation = production
)

Arguments

production

Enable production requirements.

requests_per_minute

Per-credential HTTP request ceiling.

max_rate_limit_keys

Maximum live credential/address buckets retained by one API process. Additional new identities share a bounded overflow bucket until the next minute.

trusted_proxies

Exact proxy addresses or IPv4 CIDR ranges whose 'X-Forwarded-For' header may be trusted. The default trusts none.

max_log_lines, max_log_bytes

Response ceilings for remote worker logs.

require_storage_encryption

Require 'LIBERTIES_STORAGE_KEY'.

require_os_isolation

Require independently probed container/cgroup or deployment-provided OS isolation evidence. A descriptive environment variable alone is not accepted as proof.

Value

A validated 'liberties_security_policy'.

Create a server user and one-time bearer token

Description

Only the SHA-256 digest of the cryptographically random 256-bit token is persisted. The returned token is the only copy available to the caller.

Usage

ls_user_create(
  root,
  username,
  limits = list(),
  enabled = TRUE,
  first_name = "",
  last_name = "",
  scopes = c("jobs:read", "jobs:write"),
  expires = NULL
)

Arguments

root

LibeRties server storage root.

username

Tenant identifier.

limits

Named resource limits.

enabled

Whether the account may authenticate.

first_name, last_name

Optional human-readable user names used by the administration interface. Authentication continues to use 'username'.

scopes

API permissions assigned to the token.

expires

Optional future expiry date-time. 'NULL' creates a non-expiring research token; production preflight reports it.

Value

User metadata plus the one-time token.
